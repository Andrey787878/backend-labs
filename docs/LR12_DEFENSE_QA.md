# ЛР12: вопросы для защиты

## В чём суть ЛР12?

Загрузить Excel-файл посещаемости, рассчитать посещаемость и выполненные лабораторные по каждому студенту, определить студентов, которые получают зачёт автоматически.

## Почему endpoint защищён?

По ТЗ доступ только администраторам. Используется RBAC permission `calculate-attendance`, назначенный роли `admin`.

## Как читается файл?

Через `openpyxl`. Парсер ищет лист `Посещаемость`, проверяет обязательные колонки и нормализует строки в `AttendanceRowDTO`.

## Как считается занятие?

Уникальное занятие определяется ключом:

```text
date + time + type + number
```

Группа и подгруппа в ключ занятия не входят, как указано в каверзных заметках ТЗ.

## Как считается студент?

Студенты группируются по паре:

```text
Группа + ФИО
```

Для каждого студента строится список всех уникальных занятий и отмечается `visit`.

## Что если подгруппа пустая?

Подгруппа становится `1`.

## Как считается результат?

`result = true`, если студент уже имеет `Зачёт автоматом`, либо если:

```text
visit_percent >= ATTENDANCE_PERCENT_THRESHOLD
success_labs >= REQUIRED_LABS
```

## Почему `success_labs_percent` может быть больше 100?

Если студент сделал больше лабораторных, чем требуется в `REQUIRED_LABS`, процент может быть больше 100. Это прямо разрешено в ТЗ.

## Почему поле называется `leasons`?

Так написано в примере ТЗ. Поле оставлено в таком виде, чтобы ответ соответствовал заданию.

## Где разделение SOLID?

- `AttendanceFileParser` отвечает только за Excel.
- `AttendanceCalculator` отвечает только за расчёт.
- `AttendanceResponseBuilder` отвечает только за JSON-структуру.
- `attendance_routes.py` отвечает только за HTTP/RBAC/валидацию файла.
